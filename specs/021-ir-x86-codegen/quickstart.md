# Quickstart: IR to x86-64 Code Generator

**Feature**: 021-ir-x86-codegen  
**Date**: 2025-12-16

## Overview

This guide demonstrates how to use the IR to x86-64 code generator to translate intermediate representation (IR) into executable assembly code.

---

## Basic Usage

### Step 1: Create an IR Module

```rust
use jsavrs::ir::{Module, Function, BasicBlock, DataLayout, TargetTriple};
use jsavrs::ir::types::IrType;
use jsavrs::ir::value::Value;

// Create a module for Linux x86-64
let mut module = Module::new(
    "example",
    DataLayout::LinuxX86_64,
    TargetTriple::X86_64UnknownLinuxGnu,
);

// Add a simple function
let mut func = Function::new(
    "add",
    IrType::I32,
    vec![
        IrParameter::new("a", IrType::I32),
        IrParameter::new("b", IrType::I32),
    ],
);

// ... add basic blocks and instructions ...

module.add_function(func);
```

### Step 2: Generate Assembly

```rust
use jsavrs::asm::codegen;
use jsavrs::asm::Platform;

// Generate with default options (Linux)
let assembly = codegen::generate(&module, Platform::Linux)?;

// Print the generated assembly
println!("{}", assembly);
```

### Step 3: Save and Assemble

```rust
use std::fs;

// Write to file
fs::write("output.asm", assembly.to_string())?;

// Assemble with NASM (external command)
// nasm -f elf64 output.asm -o output.o
// ld output.o -o output
```

## Platform Selection

### Linux x86-64

```rust
use jsavrs::asm::{codegen, Platform};

let assembly = codegen::generate(&module, Platform::Linux)?;
// Uses System V AMD64 ABI
// Output format: ELF64
// NASM command: nasm -f elf64 output.asm
```

### macOS x86-64

```rust
let assembly = codegen::generate(&module, Platform::MacOS)?;
// Uses System V AMD64 ABI (with symbol underscore prefix)
// Output format: Mach-O 64
// NASM command: nasm -f macho64 output.asm
```

### Windows x86-64

```rust
let assembly = codegen::generate(&module, Platform::Windows)?;
// Uses Microsoft x64 calling convention
// Output format: Win64 COFF
// NASM command: nasm -f win64 output.asm
```

## Advanced Options

### Enable Debug Comments

```rust
use jsavrs::asm::codegen::CodeGenOptions;

let options = CodeGenOptions {
    platform: Platform::Linux,
    emit_debug_comments: true,  // Include source mappings
    optimize_fall_through: true,
    collect_stats: false,
};

let (assembly, _) = codegen::generate_with_options(&module, options)?;
```

Output with debug comments:
```nasm
main:
    push rbp
    mov rbp, rsp
    sub rsp, 32
    
    ; Block: entry (from source line 10)
    ; %1 = add i32 %a, %b
    mov eax, edi          ; %a -> rdi (caller passed)
    add eax, esi          ; + %b -> rsi
    ; ...
```

### Collect Statistics

```rust
let options = CodeGenOptions {
    platform: Platform::Linux,
    emit_debug_comments: false,
    optimize_fall_through: true,
    collect_stats: true,  // Enable statistics collection
};

let (assembly, stats) = codegen::generate_with_options(&module, options)?;

println!("Functions: {}", stats.functions);
println!("Instructions: {}", stats.instructions);
println!("Spills: {}", stats.total_spills);
println!("\n{}", stats.report());
```

## Complete Example

### Input: Factorial Function in IR

```rust
use jsavrs::ir::*;
use jsavrs::ir::instruction::*;
use jsavrs::ir::types::IrType;

fn create_factorial_module() -> Module {
    let mut module = Module::new(
        "factorial",
        DataLayout::LinuxX86_64,
        TargetTriple::X86_64UnknownLinuxGnu,
    );
    
    // int64 factorial(int64 n)
    let mut func = Function::new(
        "factorial",
        IrType::I64,
        vec![IrParameter::new("n", IrType::I64)],
    );
    
    // entry:
    //   %cmp = icmp eq i64 %n, 0
    //   br %cmp, label %base, label %recurse
    let entry = BasicBlock::new("entry");
    // ... add instructions ...
    
    // base:
    //   ret i64 1
    let base = BasicBlock::new("base");
    // ... add instructions ...
    
    // recurse:
    //   %n1 = sub i64 %n, 1
    //   %r = call i64 @factorial(i64 %n1)
    //   %result = mul i64 %n, %r
    //   ret i64 %result
    let recurse = BasicBlock::new("recurse");
    // ... add instructions ...
    
    func.add_block(entry);
    func.add_block(base);
    func.add_block(recurse);
    
    module.add_function(func);
    module
}
```

### Output: Generated x86-64 Assembly

```nasm
; Assembly File - ABI: System V AMD64 ABI on Linux
; Generated by jsavrs code generator

section .text
    global factorial

factorial:
    ; Prologue
    push rbp
    mov rbp, rsp
    sub rsp, 16
    
    ; Save callee-saved register (if needed)
    push rbx
    
    ; entry:
    test rdi, rdi           ; compare n with 0
    je .base                ; if n == 0, goto base
    
    ; recurse:
    mov rbx, rdi            ; save n in rbx (callee-saved)
    dec rdi                 ; n - 1
    call factorial          ; recursive call
    imul rax, rbx           ; n * factorial(n-1)
    jmp .epilogue
    
.base:
    mov rax, 1              ; return 1
    
.epilogue:
    pop rbx
    mov rsp, rbp
    pop rbp
    ret
```

## Error Handling

```rust
use jsavrs::asm::codegen::{self, CodeGenError};

match codegen::generate(&module, Platform::Linux) {
    Ok(assembly) => {
        println!("Success! Generated {} bytes", assembly.to_string().len());
    }
    Err(CodeGenError::UnsupportedConstruct { construct, reason }) => {
        eprintln!("Cannot generate: {} - {}", construct, reason);
    }
    Err(CodeGenError::AllocationFailed { function, reason }) => {
        eprintln!("Register allocation failed in {}: {}", function, reason);
        // Consider simplifying the function or splitting it
    }
    Err(CodeGenError::StackOverflow { function, size, max }) => {
        eprintln!(
            "Stack frame too large in {}: {} bytes (max {})",
            function, size, max
        );
    }
    Err(e) => {
        eprintln!("Code generation error: {}", e);
    }
}
```

## Integration with CLI

The code generator integrates with the jsavrs CLI:

```bash
# Generate assembly from source file
jsavrs compile input.js --target x86_64-linux-gnu -o output.asm

# Generate with debug info
jsavrs compile input.js --target x86_64-linux-gnu --emit-asm --debug -o output.asm

# Cross-compile for Windows
jsavrs compile input.js --target x86_64-pc-windows-msvc -o output.asm

# Full build pipeline
jsavrs compile input.js --target x86_64-linux-gnu -o output
# This runs: codegen → nasm → ld
```

## Testing Generated Code

### Unit Test Pattern

```rust
#[cfg(test)]
mod tests {
    use super::*;
    use insta::assert_snapshot;
    
    #[test]
    fn test_factorial_codegen() {
        let module = create_factorial_module();
        let assembly = codegen::generate(&module, Platform::Linux).unwrap();
        
        // Snapshot test the generated assembly
        assert_snapshot!("factorial_linux", assembly.to_string());
    }
    
    #[test]
    fn test_cross_platform() {
        let module = create_simple_module();
        
        for platform in [Platform::Linux, Platform::MacOS, Platform::Windows] {
            let assembly = codegen::generate(&module, platform).unwrap();
            assert_snapshot!(
                format!("simple_{:?}", platform).to_lowercase(),
                assembly.to_string()
            );
        }
    }
}
```

### Integration Test with NASM

```rust
#[test]
#[ignore] // Requires NASM installed
fn test_nasm_compatibility() {
    let module = create_test_module();
    let assembly = codegen::generate(&module, Platform::Linux).unwrap();
    
    // Write assembly
    let asm_path = "target/tmp/test.asm";
    std::fs::write(asm_path, assembly.to_string()).unwrap();
    
    // Assemble with NASM
    let output = std::process::Command::new("nasm")
        .args(["-f", "elf64", asm_path, "-o", "target/tmp/test.o"])
        .output()
        .expect("NASM execution failed");
    
    assert!(output.status.success(), "NASM failed: {}", 
            String::from_utf8_lossy(&output.stderr));
}
```

## Performance Tips

1. **Minimize function complexity**: Functions with fewer than 16 live values rarely need spills
2. **Use appropriate types**: Prefer `i64` on x86-64 to match register width
3. **Enable fall-through optimization**: Default is enabled, reduces jump count by ~15%
4. **Batch module compilation**: Create one `CodeGenerator` per module to amortize setup costs
